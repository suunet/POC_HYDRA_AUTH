# T-NNN 質疑

> チャットを正本にしないための受け皿。要求の曖昧点はここでQ&Aとして確定させる。

<details>
<summary>進め方（初見時に読む）</summary>

```
AIがQを起票（未回答） → 人がAに回答し状態を「回答済み」へ → AIが受け入れ判定
   └ 解決 → 状態を「解決」にし、反映先へ反映
   └ 不明点が残る → 同じスレッドに追加のQを起票（未回答に戻す）→ 繰り返し
```

| 状態 | 意味 | 次に動く人 |
|---|---|---|
| `未回答` | AIのQに対する回答待ち | **人**（Aを書き、状態を回答済みへ） |
| `回答済み` | 人が回答済み、AIの受け入れ待ち | AI（受け入れ判定） |
| `解決` | AIが受け入れ、反映先へ反映済み | — |

- 状態を書き換えるのは: `未回答→回答済み` は**人**、`回答済み→解決` と追加Q起票（`回答済み→未回答`）は**AI**。
- 解決時は回答内容を必ず反映先（指示書・仕様カタログ等）へ反映してから「解決」にする。質疑ファイル自体は正本にしない。
- スレッドは未解決の間 `<details open>` で展開しておき、**解決したら `open` を外して畳む**。

</details>
