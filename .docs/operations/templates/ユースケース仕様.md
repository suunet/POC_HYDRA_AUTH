## 8. 実装参照（突合用）

業務・画面・HTTP・コードのどこから調査を始めても、ここを見れば同じ前提に立てる場所。**メソッド + パスを1行以上**書くことを推奨。

| 種別 | 参照 |
|---|---|
| HTTP（メソッド + パス） | 例: `POST /auth/login` |
| ルーティング | ルート登録箇所の実ファイルパス（例: `backend/<context>/module.go`） |
| Handler / UseCase / Job | 実ファイルパス（例: `backend/<context>/api/http/handler.go` ／ `backend/<context>/app/...`） |
| テスト | テストファイル（例: `backend/tests/unit/<context>_test.go`。UC-NNNでgrep可） |
| 外部連携 | EXT-NN |
| 設定・フラグ | 環境変数・設定ファイル |
