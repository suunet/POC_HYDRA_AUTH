# .docs — ドキュメント領域の案内

本プロジェクトのドキュメントは以下の区画で管理する。

## 区画一覧

| パス | 用途 | Git |
|------|------|-----|
| `.docs/design/` | 仕様の正本（仕様カタログ・ユースケース仕様・全体俯瞰図・BUC詳細） | **管理** |
| `.docs/operations/` | プロセスの正本（運用ガイド `aidd/`・ワークフロー定義・規約 `policies/`・ひな形 `templates/`） | **管理** |
| `.docs/note/` | 記録（振り返り等） | **未管理**（`.gitignore` 対象） |
| `.docs/_stash/` | ローカル作業領域（下記3区画） | **未管理**（`.gitignore` 対象） |

## `_stash/` の3区画（git未管理）

クローンしても存在しないローカル専用領域。**恒久化すべき内容をここに置き続けない**こと（下記の受け止め先へ反映して確定とする）。

| 区画 | 用途 | 恒久化の受け止め先 |
|------|------|-------------------|
| `_stash/` 直下 | 運用補足ナレッジ（ローカル限定の対応表・メモ） | 公開可能な内容が生じたら本ファイル等の管理対象ドキュメントへ |
| `_stash/share/` | 合意前のやりとり（設計案・相談ドキュメント） | 合意後に再構成して配置 |
| `_stash/tickets/T-NNN/` | チケット（指示書・質疑・レビュー報告）。チケットは**指示に徹する** | 仕様変更→ `.docs/design/`、規約変更→ `.docs/operations/`、実装→ コード。コードとの紐付けはコミットメッセージに `T-NNN` を残す |

## 原則

- **正本はGit管理の3系統:** 仕様（`.docs/design/`）・プロセス（`.docs/operations/`）・コード。チャットやチケットを正本にしない。
- `_stash/` はマシンローカルであり、バックアップ・共有されない前提で運用する。
- **Git管理ドキュメントに改訂履歴セクションは設けない**（履歴はGitで追う）。未管理ドキュメント（`_stash/` 配下）は例外的に持ってよい。
