# conditions — 条件定義（ビジネスルール）

| ID | 条件 | 関連BUC | 説明 |
|---|---|---|---|
| CND-01 | メールアドレスが未登録であること | BUC-U01 | サインアップの前提条件 |
| CND-02 | 登録済みメールアドレスに紐付くメール/パスワード認証方式が存在し、パスワードが一致すること | BUC-U04 | ログインの前提条件 |
| CND-03 | メール確認済みであること | BUC-U04 | ログインの前提条件 |
| CND-04 | アカウントが無効化済み・削除済みでないこと | BUC-U04 | ログインの前提条件 |
| CND-05 | ロックアウト中でないこと | BUC-U04 | ログインの前提条件 |
| CND-06 | アクセストークンが有効であること | BUC-U06, BUC-U08, BUC-A06, BUC-U09, BUC-U10 | 保護リソースへのアクセスの前提条件（BUC-U05はATなしRTのみで認証するため対象外） |
| CND-07 | リフレッシュトークンが有効であること | BUC-U05 | トークン再発行の前提条件 |
| CND-08 | リセットトークンが有効期限内であること | BUC-U07 | パスワードリセット完了の前提条件 |
| CND-09 | メール確認トークンが有効期限内であること | BUC-U02 | メールアドレス確認の前提条件 |
| CND-10 | 再送信時は既存の有効なメール確認トークンを無効化してから新トークンを発行すること | BUC-U03 | メール確認トークン再送信の設計制約 |
| CND-11 | 招待対象メールアドレスに管理者ロールが未付与であること | BUC-A01 | 管理者招待の前提条件（同一メールアドレスのエンドユーザー登録は許可） |
| CND-12 | 招待トークンが有効期限内であること | BUC-A02 | 招待受付の前提条件 |
| CND-13 | 対象アカウントが無効化済みであること | BUC-A05 | 管理者アカウント再有効化の前提条件 |
| CND-14 | `super_admin`が2人以上存在すること | BUC-A04, BUC-A06, BUC-A07 | `super_admin`の無効化・削除の前提条件（最低1人の`super_admin`を保持するため）。BUC-A07については降格時のみ適用（昇格は人数に関わらず許可） |
| CND-15 | 自己削除時はパスワードが一致すること | BUC-A06 | ユーザー自己削除の前提条件（不可逆操作のため本人確認が必要） |
| CND-16 | パスワード変更時は現在のパスワードが一致すること | BUC-U08 | パスワード変更の前提条件（セッション乗っ取り対策） |
| CND-17 | いずれか1つのロールが条件を満たせばアクセスを許可すること（OR評価） | 保護リソースを持つ全BUC | 複数ロール（INF-02）を持つユーザーに対するアクセス制御の共通ルール |
