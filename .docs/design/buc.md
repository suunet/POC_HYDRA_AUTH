# buc — ビジネスユースケース定義

> **ID体系**: BUC ID の `U`=利用者系 / `A`=管理者系 / `S`=システム系。`UC-NNN`（ユースケース）・`SCR-NN`（画面=APIエンドポイント）・`EVT-NN`（イベント）・`TMR-NN`（タイマー）の発番は本ファイル（フロー表の該当行）で行う。ユースケース仕様 `.docs/design/uc/UC-NNN.md` は各実装チケットのP3で作成する（UC名の洗い出し・発番は本表で完了済み）。
> **表の読み方**: 「業務」列にはBUC IDを記す（業務層は定義しない）。「次」列の「↓」はフローの向き（上から下）を示し、**すべてのアクティビティ行に付す**（関連のみの続き行には付さない）。**分岐は分岐元の「次」列にラベルを書き、同じラベルを「先」列に持つ行へ続く**。業務・BUC列が空欄の行は直上のBUCを、UC列が空欄の行は直上のUCを引き継ぐ。システムを使わないアクティビティのUC列は「—」。関連（モデル1/2）は1行につき各1つまでとし、同一アクティビティに複数の関連がある場合は**アクティビティ列を空欄にした続き行**で表す。
> **画面（SCR）・タイマー（TMR）について**: 本システムはUI無しのAPIサーバーのため、**画面=APIエンドポイント**（操作の境界）として発番する（SCR-01〜18稼働中。関連型「UC—画面—アクター」を成立させる）。未詳細化のBUC（U09・U10）はエンドポイント未設計のため画面欄なし・説明列に「未詳細化」を明記。タイマーは時間起動の処理が存在しないため当面0件（トークン有効期限は検証時に評価される条件であり、タイマー起動ではない）。
