# P1 計画（起票） — persona: DEV（定義は `start-ticket/SKILL.md`）

## 起動条件

チケット未存在（新規依頼を受けた）。

## 手順

1. チケット番号を採番する（`.docs/_stash/tickets/` の最大番号+1）。フォルダ `T-NNN/` を作り、テンプレ（`.docs/operations/templates/指示書.md`・`質疑.md`）を複製する。
2. §1（目的）・§2（要求）を起草する。
   - 要求は仕様カタログ（`.docs/design/`）の対象IDで書く。**実在しないIDを参照しない**（先にカタログをgrepして確認する）。
   - メタにチケット種別（実装／ドキュメント）・ブランチ名（`feature/t-nnn-<slug>`）を記入する。
3. 曖昧点・判断が必要な点を質疑に Q として起票する（判断材料と選択肢を添える。テンプレの進め方に従う）。
4. ステータスを `draft` として保存する。

## 禁止事項

- 要求を推測で確定させない。迷ったら質疑に出す。
- `ready-to-prepare` への変更（人間関門）。

## 停止

「T-NNN を起票しました。§1§2 と質疑を確認・回答のうえ、ステータスを `ready-to-prepare` にしてください」
