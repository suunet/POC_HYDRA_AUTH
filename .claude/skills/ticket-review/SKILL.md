# P2 要求レビュー — persona: BJ（サブエージェント）

## 起動条件

ステータス `ready-to-prepare`。

## 手順

1. 質疑を確認する。`回答済み` があれば受け入れ判定を行う（解決→反映先へ反映を確認して「解決」に更新／不明点→追加Qを起票）。`未回答` が残る場合はここで**停止**。
2. サブエージェント **BJ**（`.claude/agents/bj.md`）を「モード1: P2要求レビュー」で起動する。渡すもの: チケットパス。**自分（DEV）ではレビューしない。**
3. BJの結果を処理する:
   - **指摘ゼロ** → 何も残さず `ticket-prepare`（P3）へ連続実行。
   - **指摘あり** → `レビュー報告.md` テンプレでチケットフォルダに記録する（BJの指摘表を**改変せず**転記）。人の判断が必要な指摘は質疑に Q として起票する。
4. 上流不足（カタログ変更の要否）が指摘に含まれる場合は、その旨を停止報告で明示する（カタログ変更を本チケットで行うか別チケットに切るかは人が決める）。

## 禁止事項

- BJの指摘を握りつぶす・弱める・自分の解釈で修正する。
- 指示書§2の直接修正（修正は人の回答を受けて P1 相当で行う）。

## 停止（指摘あり時）

「BJレビューで指摘N件（レビュー報告参照）。質疑に回答してください。解消後に再度 /start-ticket を実行してください」
