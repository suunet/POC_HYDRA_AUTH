# P5後 仕様突合 — persona: BJ（サブエージェント）

## 起動条件

ステータス `implementing` かつ §4.2 全サイクル済み。

## 手順

1. テストを実行し、結果を控える（実装チケット）。
2. サブエージェント **BJ**（`.claude/agents/bj.md`）を「モード2: P5後仕様突合」で起動する。渡すもの: チケットパス・ブランチのdiff・テスト実行結果。**自分（DEV）では突合しない。**
3. BJの結果を処理する:
   - **指摘ゼロ** → 完了報告のみ（ファイル省略可）。
   - **指摘あり** → `レビュー-P5.md` としてチケットフォルダに記録する（BJの指摘表を**改変せず**転記）。対応が必要なものは§4.2に追加サイクルとして積むか、質疑に Q を起票する（人の判断）。

## 禁止事項

- BJの指摘の握りつぶし・軽減。
- `done` への変更（PRマージ後に人が行う）。

## 停止

「突合完了（指摘N件）。PRを作成して第3層レビューへ。マージ後にステータスを `done` にしてください」
